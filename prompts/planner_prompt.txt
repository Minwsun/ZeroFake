## COGNITIVE PROTOCOL: TREE OF THOUGHTS + TEMPORAL

### BRANCH 1: VERIFICATION (Event Verification)
- Goal: Prove the event OCCURRED or DID NOT occur
- Query: "[Event] site:reuters.com OR site:bloomberg.com"
- Query: "[Event] official announcement"

### BRANCH 2: TEMPORAL (Timestamp Verification)
- Goal: Determine the exact time of the event
- Query: "[Event] release date" or "[Event] when"
- Query: "[Product] launch date [year]"

### BRANCH 3: CURRENCY CHECK (Timeliness Check)
- Goal: Determine if information is still current or outdated
- Query: "[Product] latest version 2025"
- Query: "[Award] winner 2024" (most recent year)
- Query: "[Policy] current status"

---

## TACTICAL EXECUTION: GOOGLE DORKING BY CATEGORY

### Technology Products:
```
# Verify product exists
"[Product name]" site:apple.com OR site:samsung.com

# Check latest version
"iPhone" "latest" OR "newest" 2025
"Galaxy S" "latest model" 2025

# Compare versions
"iPhone 17" release date
"iPhone 15" vs "iPhone 17"
```

### Sports Events/Awards:
```
# Verify event
"[Award] [Winner] [Year]" site:fifa.com OR site:uefa.com

# Check latest award
"Ballon d'Or" winner 2024
"World Cup" champion latest
"Champions League" winner 2024
```

### Policies/Regulations:
```
# Check if still in effect
"[Policy]" "still in effect" OR "ended" OR "expired"
"Covid restrictions" Vietnam current status 2025
```

### Current Events:
```
# Check zombie news
"[Event]" date:YYYY-MM-DD (claimed date)
"[Event]" first reported when
```

---

## DETECTING WARNING SIGNALS

When analyzing a claim, watch for these signals:

| Signal | Meaning | Action |
|--------|---------|--------|
| "just now", "today" | Possible Zombie | Find original event date |
| "latest", "newest" | Possible Outdated | Find latest version/info |
| Old product (iPhone 12) | Possible Outdated | Find current product |
| Specific price/numbers | May have changed | Find current numbers |
| Covid, lockdown | Zombie News | Confirm current policy |
